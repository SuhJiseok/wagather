# WatchMe 백엔드 문서

## 서버 구성

백엔드는 `server/index.mjs` 파일에 있습니다.

사용하는 주요 구성은 아래와 같습니다.

- Express: 방 생성 API와 웹앱 제공
- Socket.IO: 실시간 입장, 채팅, 재생 상태, 참여자 상태 전달
- Vite middleware: 개발 중 프론트엔드 화면 제공

## 주요 API

### 방 만들기

```http
POST /api/rooms
```

요청 값은 아래와 같습니다.

```json
{
  "youtubeUrl": "https://www.youtube.com/watch?v=...",
  "nickname": "수민",
  "participantId": "브라우저별-참여자-ID"
}
```

응답 값은 아래와 같습니다.

```json
{
  "roomId": "ABC123",
  "inviteUrl": "/room/ABC123",
  "nickname": "수민"
}
```

## 실시간 이벤트

Socket.IO로 주고받는 주요 이벤트는 아래와 같습니다.

- `join-room`: 방 입장
- `room-state`: 현재 방 상태 전달
- `heartbeat`: 각 참여자의 현재 영상 시간과 상태 전달
- `playback-command`: 재생, 일시정지, 구간 이동 전달
- `play-countdown`: 재생 전 3초 카운트다운 전달
- `countdown-cancelled`: 카운트다운 중 정지나 이동이 발생했을 때 카운트다운 취소
- `sync-choice`: 방장 명령 이후 방 기준 시간으로 다시 합류한 상태 전달
- `chat-message`: 채팅 전달
- `emoji-reaction`: 빠른 이모지 반응 전달

## 방 상태 관리

현재 MVP는 방 정보를 서버 메모리에 저장합니다. 즉, 서버를 재시작하면 기존 방은 사라집니다.

저장하는 주요 값은 아래와 같습니다.

- 방 ID
- 유튜브 영상 ID
- 방장 ID
- 방 기준 재생 상태
- 참여자 목록
- 최근 채팅 메시지

## 동기화 방식

각 참여자는 약 0.9초마다 자신의 현재 영상 시간과 재생 상태를 서버에 보냅니다.

서버는 방 기준 시간과 각 참여자 시간을 함께 내려줍니다. 방장의 재생, 정지, 구간 이동 명령은 방 기준 상태에 저장되고 모든 참여자에게 전달됩니다.

일반 참여자가 자기 화면에서 재생, 정지, 구간 이동을 해도 서버의 방 기준 상태는 바뀌지 않습니다.

## 재생 카운트다운 방식

방장이 `playback-command`의 `play`를 보내면 서버는 첫 재생 때 즉시 재생 상태로 바꾸지 않습니다.

서버는 현재 방 기준 시간을 일시정지 상태로 고정하고, 모든 참여자에게 `play-countdown` 이벤트를 보냅니다. 3초 뒤 서버가 방 기준 상태를 `playing`으로 바꾸고 `remote-command`를 보내 모두 같은 기준 시간에서 재생하게 합니다.

카운트다운 도중 정지나 구간 이동 명령이 들어오면 기존 카운트다운을 취소하고 `countdown-cancelled` 이벤트를 보냅니다.

일반 참여자의 재생 명령은 서버에서 거부됩니다. 서버는 방장의 명령만 방 기준 상태에 반영합니다.

## 현재 제한 사항

- 서버 재시작 시 방이 사라집니다.
- 데이터베이스가 없습니다.
- 방별 최대 인원 제한은 코드로 강제하지 않았습니다.
- 광고를 직접 감지하거나 제거하지 않습니다.
- 유튜브 외 영상 플랫폼은 지원하지 않습니다.
