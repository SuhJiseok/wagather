# WatchMe 프론트엔드 문서

## 화면 구성

현재 프론트엔드는 React로 구성되어 있습니다.

주요 화면은 3개입니다.

- 홈 화면: 유튜브 링크와 닉네임을 입력해 방을 만듭니다.
- 입장 화면: 초대 링크로 들어온 사용자가 닉네임을 입력합니다.
- 같이 보기 방: 유튜브 영상, 채팅, 이모지, 참여자 상태를 보여줍니다.

## 주요 파일

- `src/main.tsx`: 앱 시작점입니다.
- `src/App.tsx`: 화면과 주요 동작이 들어 있습니다.
- `src/styles.css`: 전체 디자인과 반응형 스타일이 들어 있습니다.
- `src/vite-env.d.ts`: 유튜브 플레이어 타입 정의가 들어 있습니다.

## 디자인 방향

앱 화면은 일반적인 카드형 대시보드보다, 영상 시청 화면에 가까운 어두운 톤의 워크스페이스로 구성했습니다.

- 배경은 어둡고 차분하게 유지합니다.
- 주요 액션은 밝은 라임색으로 강조합니다.
- PC에서는 영상이 왼쪽, 채팅/참여자는 오른쪽에 위치합니다.
- 모바일에서는 영상 아래에 `채팅`과 `참여자` 탭을 둡니다.
- 모바일 방 화면은 `헤더`, `영상`, `방 상태`, `채팅/참여자`가 한 화면에 최대한 들어오도록 압축합니다.
- 모바일 방 화면에서는 브라우저 전체 스크롤을 잠그고, 입력창 포커스 시 영상이 화면 밖으로 밀리지 않게 유지합니다.
- 유튜브 플레이어 위에는 앱 UI를 덮지 않습니다.

## 개인 조작 정책

일반 참여자가 유튜브 플레이어에서 재생, 정지, 구간 이동을 해도 해당 조작은 본인 화면에만 적용됩니다. 이 동작은 서버의 방 기준 재생 상태를 바꾸지 않습니다.

일반 참여자가 개인 조작을 하면 화면 상태가 `개인 조작 중`으로 표시됩니다. 사용자는 `현재 방 시간으로 맞추기`를 눌러 방장 기준 시간으로 직접 돌아올 수 있습니다.

방장이 재생, 정지, 구간 이동을 하면 방장 명령이 모든 참여자에게 전달되고, 일반 참여자가 개인적으로 보고 있던 화면도 방장 기준 시간으로 자동으로 맞춰집니다.

## 재생 카운트다운 UI

방장이 처음 영상을 재생하면 즉시 재생하지 않고 팝업으로 3초 카운트다운을 보여줍니다.

```text
곧 같이 재생돼요
3
```

카운트다운이 끝나면 방 기준 시간에서 영상이 재생됩니다.

## 상태 표시

방 화면은 참여자 수와 방장 표시를 보여줍니다. 방장은 전체 재생 권한을 가진 사람으로 표시되고, 일반 참여자는 로컬 조작을 해도 다른 사람에게 영향을 주지 않습니다. 개인 조작 중인 사용자의 본인 화면에는 `개인 조작 중` 상태가 표시됩니다.

## 실행과 확인

```bash
npm install
npm run dev
```

접속 주소는 아래입니다.

```text
http://localhost:3000
```

프론트엔드 확인 시 봐야 할 항목은 아래와 같습니다.

- 홈 화면 입력칸과 버튼이 모바일/PC에서 깨지지 않는지
- 방 화면에서 유튜브 영상과 채팅이 겹치지 않는지
- 모바일 방 화면에서 주요 영역이 스크롤 없이 한 화면에 들어오는지
- 채팅 입력창을 눌러도 영상이 화면 밖으로 밀리지 않는지
- 모바일에서 채팅/참여자 탭 전환이 되는지
- 방장이 처음 재생을 눌렀을 때 3초 카운트다운이 뜨는지
- 방장의 정지, 재생, 구간 이동이 다른 참여자에게 반영되는지
- 일반 참여자의 로컬 조작이 다른 사람에게 반영되지 않는지
- `현재 방 시간으로 맞추기` 버튼으로 개인 조작 상태에서 방 시간으로 돌아올 수 있는지
