# WatchMe

멀리 떨어져 있는 친구나 연인이 같은 유튜브 영상을 함께 보면서, 같은 화면 안에서 채팅과 반응을 주고받는 웹 서비스 MVP입니다.

## 현재 구현 상태

- 유튜브 링크로 같이 보기 방 만들기, 초대 링크로 입장, 닉네임 기반 참여
- 방장이 처음 재생을 누르면 3초 카운트다운 후 같이 재생, 정지·구간 이동도 동기화
- 일반 참여자의 재생·정지·구간 이동은 본인 화면에만 적용(개인 조작 중) + 방 시간 재합류
- 실시간 채팅, 빠른 이모지 반응, 영상 위 위치 지정 이모지 반응
- 영상 위 커서챗 말풍선(`/` 키, PC 전용)과 탭 핑
- PC용 커스텀 플레이어 컨트롤(재생/탐색/음량/전체화면), 모바일은 유튜브 네이티브 컨트롤
- 인기 영상 둘러보기(카테고리·길이 필터, 무한 스크롤), 시청 히스토리, 이어보기 지점
- 참여자 목록과 상태(방장/싱크 모드) 표시
- 새로고침 시 마지막 위치 복원(브라우저 자동재생 정책 대응: 음소거로 시작하고 한 번의 탭으로 소리)

## 실행 방법

```bash
npm install
npm run dev
```

실행 후 브라우저에서 아래 주소로 접속합니다.

```text
http://localhost:3000
```

## 유튜브 메타데이터

일반 사용자는 유튜브 링크만 입력하면 됩니다. 영상 카테고리까지 가져오려면 서버 환경변수에 YouTube Data API 키를 등록합니다.

```bash
cp .env.example .env
```

```env
YOUTUBE_API_KEY=your_youtube_data_api_key
YOUTUBE_CATEGORY_REGION=KR
YOUTUBE_CATEGORY_LANGUAGE=ko
YOUTUBE_POPULAR_MAX_RESULTS=24
```

API 키가 있으면 실제 YouTube 인기 영상 차트와 카테고리별 인기 영상을 가져옵니다. 카테고리 필터 목록은 서버의 고정값을 사용합니다. 인기영상 페이지는 스크롤 시 다음 묶음을 이어 불러오며, `YOUTUBE_POPULAR_MAX_RESULTS`는 한 번에 가져오는 영상 수입니다. API 키가 없거나 요청에 실패하면 기존처럼 YouTube oEmbed 또는 내장 샘플 목록으로 대체합니다.

## 참고

첫 버전은 유튜브 광고를 제거하거나 직접 제어하지 않습니다. 유튜브 플레이어 안에서 광고가 재생되는지는 사용자별 계정, 프리미엄 여부, 브라우저 상황에 따라 달라질 수 있으므로, 현재 MVP는 시간 차이를 감지해 사용자에게 선택지를 보여주는 방식으로 대응합니다.

## 배포

### Render (실시간 서버 포함 — 권장)

저장소 루트의 `render.yaml` 블루프린트로 Express + Socket.IO 서버 전체가 배포됩니다. 빌드된 프론트도 같은 서버가 서빙하므로 별도 프론트 배포가 필요 없습니다.

1. [render.com](https://render.com) 가입 후 GitHub 계정 연결
2. 대시보드에서 **New + → Blueprint** 선택
3. `wagather` 저장소 선택 → `render.yaml`이 자동 인식됨 → **Apply**
4. 배포가 끝나면 `https://wagather.onrender.com` 형태의 주소가 발급됩니다 (이 주소로 모바일에서 실시간 같이 보기 동작 확인)

이후에는 `main` 브랜치에 푸시할 때마다 자동으로 재배포됩니다.

참고: 무료 플랜은 15분간 요청이 없으면 서버가 잠들어, 첫 접속 시 깨어나는 데 30초~1분 정도 걸릴 수 있습니다. 방 데이터는 서버 메모리에만 있으므로 재배포/재시작 시 초기화됩니다.

### Vercel (정적 프론트 미리보기)

Vercel 배포는 모바일 화면 확인을 위한 정적 프론트 미리보기 용도입니다. Vercel에서는 Express/Socket.IO 실시간 서버가 돌지 않기 때문에, 방 생성 시 `프론트 미리보기` 방으로 이동합니다. 실시간 기능 확인은 Render 주소나 로컬 개발 서버를 사용하세요.
